# Data Models & Schemas

The application enforces standard JSON schemas defined via Pydantic v2. To survive inconsistencies in local LLM outputs (which frequently omit fields or return explicit `null` fields), the system utilizes a custom `SafeBaseModel` base class.

---

## 1. SafeBaseModel (Null-Handling)

The `SafeBaseModel` utilizes a `before` model validator to preprocess raw JSON inputs before Pydantic evaluates them:
* Any key whose value is explicitly returned as `null` or `None` is mapped to its matching default type.
* Lists resolve to `[]` instead of raising a validation error.
* Dictionaries resolve to `{}`.
* Strings resolve to `""`.
* Numbers (int/float) resolve to `None` or `0`.

This prevents errors like:
`Input should be a valid string [type=string_type, input_value=None]`

## 2. Core Schemas

### 2.1 ExperienceEntry
Defines a historical position inside the candidate's work history.
* `company: str = ""`
* `role: str = ""`
* `start_date: str = ""`
* `end_date: Optional[str] = None`
* `description: str = ""`
* `achievements: list[str] = []`

### 2.2 EducationEntry
Defines an academic qualification.
* `institution: str = ""`
* `degree: str = ""`
* `field: str = ""`
* `graduation_year: Optional[int] = None`

### 2.3 ResumeAnalysis
The structured profile extracted by the Resume Analyst agent:
* `candidate_name: Optional[str] = None`
* `contact_info: dict = {}`
* `professional_summary: Optional[str] = None`
* `skills: list[str] = []`
* `experience: list[ExperienceEntry] = []`
* `education: list[EducationEntry] = []`
* `ats_readability_score: int` (0-100)
* `strengths: list[str] = []`
* `weaknesses: list[str] = []`
* `improvement_suggestions: list[str] = []`

### 2.4 JobAnalysis
The parsed vacancy specifications generated by the Job Analyst agent:
* `job_index: int`
* `title: str = ""`
* `company: Optional[str] = None`
* `seniority_level: str = "mid"`
* `required_skills: list[str] = []`
* `desired_skills: list[str] = []`
* `ats_keywords: list[str] = []`
* `years_experience_required: Optional[int] = None`
* `key_responsibilities: list[str] = []`
* `compatibility_score: int`
* `gap_analysis: str = ""`

### 2.5 OptimizedResume
The final output of the optimizer:
* `job_index: Optional[int] = None`
* `target_job_title: str = ""`
* `content: ResumeContent` (Nested summary, skills, experience, education, certifications, and additional sections)
* `changes_made: list[str] = []`
* `keywords_added: list[str] = []`
* `estimated_ats_score: int`
* `compatibility_score: int`
